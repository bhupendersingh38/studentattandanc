  📊 DATABASE STRUCTURE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Table: users
   • Login accounts for admin, teacher, student
   • Fields: id, user_id, email, password_hash, role

Table: students
   • Complete student profiles
   • Fields: roll_number, name, email, phone, parent_phone
   • semester, class_section, department, photo_path
   • attendance_percentage, total_classes, attended_classes

Table: teachers
   • Teacher profiles
   • Fields: teacher_id, name, email, phone, department

Table: attendance_records
   • Every attendance entry
   • Fields: student_id, teacher_id, date, time, subject
   • confidence_score, liveness_passed, distance_verified
   • verification_status (pending/approved/rejected)

Table: notifications
   • SMS/Email notifications
   • Fields: user_id, title, message, type, status

Table: timetables
   • Class schedules
   • Fields: semester, class_section, day, time_slot, subject

HEALTH & STATUS:
   GET  /health              - Check if backend is running
   GET  /ready               - Readiness check
   GET  /docs                - API documentation (Interactive!)

STUDENTS:
   GET    /api/v1/students                  - Get all students (with filters)
   GET    /api/v1/students/{id}             - Get single student
   POST   /api/v1/students                  - Register new student
   POST   /api/v1/students/{id}/photo       - Upload student photo
   PUT    /api/v1/students/{id}             - Update student
   DELETE /api/v1/students/{id}             - Delete student

ATTENDANCE:
   POST   /api/v1/attendance/mark           - Mark attendance
   GET    /api/v1/attendance/today          - Today's attendance
   GET    /api/v1/attendance/student/{id}   - Student history
   POST   /api/v1/attendance/verify         - Teacher verification

ANALYTICS:
   GET    /api/v1/analytics/dashboard       - Dashboard stats
   GET    /api/v1/analytics/reports         - Generate reports
   GET    /api/v1/analytics/at-risk         - At-risk students

AUTHENTICATION:
   POST   /api/v1/auth/login                - User login
   POST   /api/v1/auth/register             - User registration

1. Register Student (Python):

import requests

data = {
    "roll_number": "21CS001",
    "name": "Rahul Kumar",
    "email": "rahul@example.com",
    "phone": "+91 9876543210",
    "parent_phone": "+91 9876543211",
    "semester": 1,
    "class_section": "A",
    "department": "CSE",
    "dob": "2003-01-01",
    "address": "123 Main St"
}

response = requests.post("http://localhost:8000/api/v1/students", json=data)
print(response.json())

# Output:
{
    "message": "Student registered successfully",
    "student_id": 1,
    "credentials": {
        "user_id": "21CS001",
        "password": "Std@C001"
    }
}


2. Upload Photo:

with open("student.jpg", "rb") as f:
    files = {"file": f}

3. Get Students (with filters):

# All students

    "http://localhost:8000/api/v1/students?semester=1&class_section=A"
)

# Search by name
response = requests.get(
    "http://localhost:8000/api/v1/students?search=Rahul"
)

Student Registration:
   Teacher fills form
   → POST /api/v1/students
   → Data saved to database
   → User account created
   → Credentials generated
   → Return to teacher
   → SMS sent to student & parent

Photo Upload:
   Teacher uploads photo
   → POST /api/v1/students/{id}/photo
   → Saved to uploads/student_photos/
   → Path stored in database
   → Face encoding generated (AI)
   → Ready for recognition

Mark Attendance:
   Student opens camera
   → Face detected
   → Compared with database
   → Liveness checked
   → Distance verified
   → POST /api/v1/attendance/mark
   → Saved to database
   → Teacher notification
   → Pending verification

Teacher Verification:
   Teacher reviews attendance
   → Checks confidence score
   → Checks liveness status
   → Approves or rejects
   → Status updated in database
   → SMS sent to parent
   → Statistics updated
